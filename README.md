# 📋 Task Manager API + Telegram Bot

Полноценный pet-проект для управления задачами, состоящий из REST API на **FastAPI** и **Telegram-бота** на **Aiogram 3**.

Проект разрабатывался не как учебный CRUD, а как возможность познакомиться с современным стеком Backend-разработки и научиться объединять несколько сервисов в одно приложение.

### При разработке использовался ИИ как вспомогательный инструмент (поиск ошибок, рефакторинг отдельных участков кода и автоматизация рутинных задач). Вся архитектура, логика приложения и реализация функциональности были разработаны самостоятельно.

# Почему используется SQLite?

В проекте используется SQLite вместо PostgreSQL.

Это сделано сознательно.

Цель проекта заключалась не в изучении конкретной СУБД, а в освоении SQLAlchemy, Alembic, архитектуры приложения и взаимодействия компонентов между собой.

Благодаря использованию ORM переход на PostgreSQL потребует минимальных изменений.

---

# Почему используется Redis?

В проекте он выполняет реальные задачи:

* хранение JWT-токенов пользователей;
* контроль времени жизни токена (TTL);
* хранение состояний FSM Telegram-бота.

Была возможность использовать MemoryStorage вместо Redis, но мне так больше понравилось ) 

---

# Почему именно такой проект?

Мне хотелось выйти за рамки обычного CRUD-приложения.

Поэтому я объединил несколько технологий в одном проекте:

* REST API;
* Telegram Bot;
* Redis;
* JWT;
* Docker;
* тестирование;
* GitHub Actions.

Благодаря этому проект получился максимально приближенным к реальному backend-приложению.

## 🚀 Запуск проекта

Проект запускается с помощью **Docker Compose**.

В репозитории есть две конфигурации:

* **docker-compose.yml** — основная конфигурация для запуска приложения, приближенная к production.
* **docker-compose.dev.yml** — конфигурация для разработки с автоматическим подхватом изменений в коде без постоянной пересборки Docker-образов.
